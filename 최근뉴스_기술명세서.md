# 최근 뉴스 조회 및 읽어주기 기능 기술 명세서

기존 알람 웹앱(React)에 "최신 뉴스 5개 검색 및 음성 읽어주기" 기능을 **Supabase + React** 스택으로 구현하기 위한 기술 검토 문서입니다.

## 1. 전체 아키텍처

```
[React 프론트엔드]
   ├─ 알람 기능: setInterval + Notification API + Audio (기존 UI 그대로 활용)
   ├─ 뉴스 표시: Supabase Edge Function 호출 → 뉴스 5개 리스트 렌더링
   ├─ 읽어주기: Web Speech API로 뉴스 제목/본문 낭독
   └─ 인증/데이터: Supabase Auth + Supabase Client SDK

[Supabase]
   ├─ Edge Functions (Deno 런타임)
   │    └─ 네이버 뉴스 API 또는 RSS 호출 → CORS 우회 + API 키 보호 → JSON 반환
   ├─ Database (Postgres)
   │    ├─ alarms 테이블: 사용자별 알람 목록 저장
   │    └─ news_cache 테이블 (선택): 뉴스 결과 캐싱으로 API 호출 절약
   ├─ Auth
   │    └─ 사용자별 알람/뉴스 설정 저장 시 로그인 기능 필요하면 사용
   └─ Realtime (선택)
        └─ 여러 기기에서 알람 동기화하고 싶을 때 활용 가능
```

## 2. 뉴스 데이터 가져오기 — Supabase Edge Functions 활용

브라우저에서 직접 뉴스 API(네이버, Google News RSS 등)를 호출하면 CORS 문제와 API 키 노출 문제가 생깁니다. 이를 Supabase의 **Edge Functions**로 해결합니다.

### 동작 방식
1. React 앱에서 `supabase.functions.invoke('get-news')` 호출
2. Edge Function(Deno)이 서버 사이드에서 네이버 뉴스 검색 API 또는 Google News RSS를 호출
3. 결과를 정리해서 JSON으로 5개 뉴스 반환
4. API 키(Client ID/Secret)는 Supabase 프로젝트의 **환경 변수(Secrets)**에 저장 → 브라우저에는 노출되지 않음

### 예시 코드 (Edge Function, `supabase/functions/get-news/index.ts`)
```typescript
import { serve } from "https://deno.land/std/http/server.ts";

serve(async (req) => {
  const clientId = Deno.env.get("NAVER_CLIENT_ID");
  const clientSecret = Deno.env.get("NAVER_CLIENT_SECRET");

  const res = await fetch(
    "https://openapi.naver.com/v1/search/news.json?query=속보&display=5&sort=date",
    {
      headers: {
        "X-Naver-Client-Id": clientId!,
        "X-Naver-Client-Secret": clientSecret!,
      },
    }
  );
  const data = await res.json();

  return new Response(JSON.stringify(data.items), {
    headers: { "Content-Type": "application/json", "Access-Control-Allow-Origin": "*" },
  });
});
```

### 프론트엔드 호출 예시 (React)
```javascript
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

async function fetchNews() {
  const { data, error } = await supabase.functions.invoke('get-news');
  if (error) console.error(error);
  return data; // 뉴스 5개 배열
}
```

### 옵션 비교
| 뉴스 소스 | 장점 | 단점 |
|---|---|---|
| 네이버 뉴스 검색 API | 한국어 뉴스 최적화, 무료 | Client ID/Secret 필요 (Edge Function으로 해결) |
| Google News RSS | API 키 불필요 | CORS 문제 (Edge Function으로 해결), 파싱 로직 필요 |
| NewsAPI.org | 사용 쉬움 | 무료 플랜은 localhost 전용, 배포 불가 |

→ **네이버 뉴스 검색 API + Supabase Edge Function** 조합을 1순위로 추천합니다.

## 3. 뉴스 읽어주기 (TTS)

- **Web Speech API (`SpeechSynthesisUtterance`)**: 브라우저 내장 기능이라 별도 서버·비용 없이 바로 사용 가능. `lang: 'ko-KR'`로 지정하면 한국어 음성 지원. React 컴포넌트 안에서 바로 호출 가능한 가장 간단한 방법.

```javascript
function readNews(text) {
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = 'ko-KR';
  speechSynthesis.speak(utterance);
}
```

- **더 자연스러운 음성이 필요하면**: 네이버 Clova Voice, Google Cloud TTS 같은 클라우드 TTS API를 Supabase Edge Function을 통해 호출 (API 키를 서버 쪽에 보호). 음질은 좋지만 과금이 발생하므로 초기에는 Web Speech API로 시작하는 것을 권장합니다.

## 4. 데이터베이스 설계 (선택 사항)

알람뿐 아니라 사용자별 뉴스 설정(관심 키워드, 읽어주기 여부 등)도 저장하고 싶다면 Supabase Database(Postgres) 테이블을 아래처럼 구성할 수 있습니다.

```sql
-- 알람 테이블
create table alarms (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users(id),
  time time not null,
  label text,
  repeat_days text[], -- 예: {'일','월','화'}
  created_at timestamp default now()
);

-- 뉴스 캐시 테이블 (선택, API 호출 절약용)
create table news_cache (
  id uuid primary key default gen_random_uuid(),
  fetched_at timestamp default now(),
  news_json jsonb
);
```

- **Row Level Security(RLS)**를 켜서 사용자가 본인 알람만 조회/수정하도록 설정 권장
- 비로그인 상태로 로컬 알람만 쓰고 싶다면 Auth/DB 없이 Edge Function(뉴스 기능)만 사용해도 무방

## 5. 인증 (선택 사항)

- 여러 기기에서 알람을 동기화하거나 사용자별 뉴스 관심사를 저장하려면 **Supabase Auth** 사용
- 이메일/비밀번호, 소셜 로그인(Google, Kakao 등) 모두 지원
- 간단한 개인용 알람앱이라면 Auth 없이 로컬 상태(useState/localStorage 대체로 Supabase 익명 사용자 등) 활용도 가능

## 6. 요약 추천 스택

| 기능 | 추천 기술 |
|---|---|
| 프론트엔드 | React (기존 알람 UI 그대로 확장) |
| 뉴스 조회 프록시 | Supabase Edge Functions (Deno) |
| 뉴스 소스 | 네이버 뉴스 검색 API |
| 음성 낭독 | Web Speech API (`ko-KR`) |
| 데이터 저장 | Supabase Database (Postgres) — 알람/설정 저장 |
| 인증 (선택) | Supabase Auth |
| 배포 | Vercel/Netlify(프론트) + Supabase(백엔드) 조합 무료 티어로 충분 |
