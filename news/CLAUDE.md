# news/CLAUDE.md

이 파일은 `news` 폴더 안의 파일을 다룰 때만 참고하는 세부 규칙이다. 프로젝트 전체 공통 규칙(보안 등)은 루트 `CLAUDE.md`를 따르며, 여기서는 반복하지 않는다.

## 담당 범위

이 폴더는 알람 앱의 "최근뉴스" 조회 및 TTS(음성) 브리핑 기능만 담당한다.

## 파일 구조 규칙

- 이 폴더에는 `news.js`, `news.css` 파일만 둔다. 그 외 파일을 새로 만들지 않는다.

## 함수 이름 규칙

- 데이터를 가져오는 함수: `fetchXxx` (예: `fetchNews`)
- 화면에 그리는 함수: `renderXxx`
- 순수 변환 함수(입출력만 있고 부수효과 없음): `buildXxx`, `stripXxx`

## 함수 시그니처 규칙

> 참고: `최근뉴스_기술명세서.md`에는 5번/9.4번 같은 별도 함수 시그니처 표가 없다. 문서 내에서 시그니처가 실제로 존재하는 함수는 2번 섹션의 `fetchNews()`와 3번 섹션의 `readNews(text)` 두 가지뿐이므로, 이 두 예시 코드를 시그니처 기준으로 삼는다.

- `fetchNews()`: 파라미터 없이 호출하며, 뉴스 5개 배열을 반환한다. 명세서 예시 코드 그대로의 이름·파라미터 구조를 따르고 임의로 바꾸지 않는다.
- `readNews(text)`: 파라미터 이름 `text`를 그대로 사용하며, 내부에서 `SpeechSynthesisUtterance`의 `lang`을 `'ko-KR'`로 고정한다.
- 명세서에 시그니처가 명시되지 않은 새 함수를 추가해야 할 경우, 임의로 확정하지 말고 사용자에게 먼저 확인한다.

## script.js와의 경계

- `script.js`(알람 로직)는 이 폴더의 함수를 호출만 하고, 뉴스 조회·렌더링·TTS 관련 로직을 직접 작성하지 않는다.
- 뉴스 관련 로직이 필요하면 `news.js`에 함수를 추가하고, `script.js`에서는 해당 함수를 호출하는 코드만 작성한다.
